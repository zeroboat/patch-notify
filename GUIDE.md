## 1. 로그인 및 화면 구성

로그인 후 대시보드가 표시됩니다. 왼쪽 사이드바에서 원하는 메뉴로 이동합니다.

**사이드바 메뉴 구성**

**Dashboards**

| 메뉴 | 설명 | 접근 권한 |
|------|------|----------|
| Dashboards | 요약 현황 (고객사 수, 이번 달 패치노트, 최근 패치노트) | 전체 |
| Patch Notes | 솔루션별 제품 및 유틸리티 패치노트 목록 | 전체 |
| Official Notice | 공문 작성 및 발송 | Admin, Manager |
| Logs | 이메일/Slack 발송 이력 | Admin, Manager, SE |
| Feedback | 피드백 목록 | Guest 제외 |

**Management**

| 메뉴 | 설명 | 접근 권한 |
|------|------|----------|
| Product Management | 솔루션 및 제품 관리 | Admin, Manager |
| Utilities Management | 유틸리티 관리 | Admin, Manager |
| Customer Management | 고객사 등록 및 이메일 관리 | Guest 제외 |
| Notice Settings | 공문 이메일 템플릿 설정 | Admin, Manager |
| Notion Integration | Notion 페이지 동기화 | Admin, Manager |
| Slack Workspaces | Slack 워크스페이스 승인 및 고객사 연결 | Admin, Manager |
| Action Log | 주요 액션 이력 조회 | Admin, Manager |
| Subscribers | 고객사별 구독 설정 | Guest 제외 |

> 역할에 따라 보이는 메뉴가 다릅니다.

## 2. 패치노트 작성하기

**권한:** Admin, Manager, Dev

1. 왼쪽 사이드바에서 **솔루션 > 제품명** 클릭
2. 우측 상단 **패치노트 추가** 버튼 클릭
3. 버전과 배포일 입력
4. 아래 항목에 내용 입력:

| 항목 | 설명 |
|------|------|
| 기능 추가 | 이번 버전에서 새로 추가된 기능 |
| 기능 개선 | 기존 기능이 변경되거나 향상된 내용 |
| 버그 수정 | 수정된 버그 목록 |
| 특이사항 | 주의사항, 마이그레이션 안내 등 |
| 내부 사항 | 외부 발송에 포함되지 않는 내부 메모 |

5. 각 항목에서 **+ 추가** 버튼으로 항목 추가, 하위 항목 버튼으로 들여쓰기 항목 추가 가능
6. **저장** 클릭

> Ollama 번역 서버가 연결되어 있으면 저장 후 영문 번역이 자동으로 시작됩니다.
> 기능 추가, 기능 개선, 버그 수정의 경우 기존과 동일하게 CodeBlock안에 작성되기 때문에 평문으로 작성 부탁드립니다.

## 3. 유틸리티 패치노트

**권한:** Admin, Manager, Dev

유틸리티는 특정 솔루션에 속하지 않는 독립 도구(CLI, 플러그인 등)의 패치노트를 관리합니다. 플랫폼(iOS / Android / Web / 공통) 단위로 분류됩니다.

**경로:** 사이드바 > Patch Notes > Utilities > 유틸리티명

### 유틸리티 등록

1. 사이드바 Management > **Utilities Management** 클릭
2. **유틸리티 추가** 버튼 클릭
3. 이름, 플랫폼(iOS/Android/Web/공통), 표시 순서 입력
4. 다운로드 파일 필수 여부 체크
   - 활성화 시 Release 파일을 업로드해야만 발행 가능
5. **저장**

### 유틸리티 패치노트 작성

제품 패치노트와 동일한 방식으로 작성합니다 ([2. 패치노트 작성하기](#2-패치노트-작성하기) 참고).

**유틸리티 전용 Notion Push 항목**

| 항목 | 내용 |
|------|------|
| 지원 종료 예정 | 릴리즈일로부터 1년 후 날짜 자동 계산 |
| Download | 첨부된 Release 파일의 NAS 다운로드 링크 자동 삽입 |

## 4. 패치노트 수정 및 삭제

- **수정:** 패치노트 항목 클릭 → 내용 수정 → **저장**
- **삭제:** 패치노트 우측 삭제 버튼 클릭 → 확인

> 이미 발행된 패치노트는 수정해도 기존에 발송된 알림이 다시 전송되지 않습니다.

## 5. 파일 첨부하기

**권한:** Admin, Manager, Dev

패치노트에 배포 파일(APK, AAR, Framework 등)을 첨부할 수 있습니다.

1. 패치노트 상세 화면 하단 **파일 첨부** 영역으로 이동
2. 파일 유형 선택:
   - **Release** — 정식 배포 파일
   - **Debug** — 디버그 정보 포함 파일
3. 파일 선택 후 **업로드**
4. Nextcloud(NAS)가 연결된 경우 자동으로 이중 저장되며 공유 링크가 생성됩니다.

## 6. 패치노트 발행하기

**권한:** Admin, Manager, Dev

> ⚠️ 발행하면 즉시 알림이 발송됩니다. 내용을 꼭 확인하고 발행하세요.

1. 발행할 패치노트 우측 **발행** 버튼 클릭
2. 확인 다이얼로그에서 **발행** 클릭
3. 발행이 완료되면 구독 중인 고객사에 알림이 자동 발송됩니다:
   - **Gmail** 구독자 → 이메일로 전송
   - **Slack** 구독자 → 지정된 채널로 메시지 전송

**발송되는 Slack 메시지 예시**

```
[AppSuit Android Library Release 안내]

Version: 1.2.3  ·  2025-01-15
──────────────────────────────
기능 추가
  - 신규 API 지원
버그 수정
  - 크래시 수정
```

발송 결과는 **발송 로그** 메뉴에서 확인할 수 있습니다.

## 7. 고객사 관리하기

**권한:** Admin, Manager, SE

**경로:** 사이드바 > 고객사 관리

### 고객사 등록

1. **고객사 추가** 버튼 클릭
2. 고객사명 입력
3. On-Premise 여부 체크 (해당되는 경우)
4. 구매 솔루션 선택 (복수 선택 가능)
5. **저장**

### 수신 이메일 추가

1. 고객사 카드 내 **이메일 추가** 클릭
2. 이메일 주소와 담당자명(선택) 입력
3. **추가**

> 여기서 등록하는 이메일은 **공문(Official Notice) 수신 전용**입니다. 패치노트 Gmail 구독 수신 이메일은 Subscribers 메뉴에서 별도로 관리합니다.

### Google 연락처 CSV로 일괄 추가

1. Google 연락처에서 CSV 내보내기
2. **CSV 가져오기** 버튼으로 파일 업로드
3. 이메일 자동 등록

## 8. 구독 설정하기

**권한:** Admin, Manager, SE

**경로:** 사이드바 > Subscribers

고객사별로 패치노트 알림 구독을 관리합니다.

각 고객사 행의 Gmail / Slack 열은 `구독 솔루션 수 / 구매 솔루션 수` 형식으로 표시됩니다. 뱃지를 클릭하면 구독 중인 솔루션 목록을 드롭다운으로 확인할 수 있습니다.

### 관리자 구독 설정

제품 단위 세부 설정은 관리자가 직접 구성합니다.

1. 고객사 행 우측 **설정** 버튼 클릭
2. 솔루션 > 제품별로 **Gmail** / **Slack** 활성화 여부 설정
3. **최대 수신 건수** 설정 (1~10건)
   - 발행 시 최근 N건의 패치노트를 묶어서 전송
4. Slack 알림의 경우 **채널 ID** 입력
   > **채널 ID 확인 방법:** Slack 채널명 우클릭 → 채널 정보 보기 → 하단에 채널 ID 표시
5. **저장**

### 수신 이메일 관리

패치노트 Gmail 알림을 수신할 이메일 주소를 관리합니다. 공문 수신 이메일(고객사 관리)과 별도로 운영됩니다.

1. 고객사 행 우측 **봉투 아이콘 버튼** 클릭
2. 등록된 수신 이메일 목록 확인
3. 이메일 주소와 담당자명(선택) 입력 후 **추가**
4. 삭제하려면 해당 행의 휴지통 아이콘 클릭

> 고객사가 구독 링크를 통해 직접 추가/삭제한 이메일도 이 목록에 반영됩니다.

### 구독 링크 발행

고객사에게 링크를 발행하면 고객사가 직접 구독 설정을 관리할 수 있습니다.

1. 고객사 행 우측 **링크 버튼(🔗)** 클릭
2. **만료일** 입력 (기본값: 30일 후)
3. **링크 발행** 클릭
4. 발행 완료 후 모달 내 링크를 복사하여 고객사에 전달
5. 구독 링크 열에 만료일까지 남은 일수 표시:
   - `D-N` 초록 뱃지: 정상
   - `D-N` 주황 뱃지: 만료 7일 이내
   - 회색 뱃지: 만료됨

> **재발행** 시 기존 링크는 즉시 무효화되고 새 링크가 생성됩니다. **링크 취소**로 즉시 무효화할 수 있습니다.

### 고객사 구독 페이지 (공개)

고객사가 전달받은 링크에 접속하면 로그인 없이 아래 작업을 수행할 수 있습니다.

| 기능 | 설명 |
|------|------|
| Gmail 알림 구독 토글 | 구매한 솔루션별로 이메일 알림 활성화 / 비활성화 |
| 수신 이메일 추가 | 이메일 주소와 담당자명 입력 후 추가 |
| 수신 이메일 삭제 | 등록된 이메일 주소 삭제 |

> 모든 변경사항은 즉시 저장됩니다. 링크 만료 후에는 접근이 불가합니다.

## 9. 공문 발송하기

**권한:** Admin, Manager

**경로:** 사이드바 > 공문 발송

패치노트 알림과 별개로 공지사항이나 안내문을 이메일로 발송합니다.

### 발송 방법 선택

**솔루션 선택 방식**
- 솔루션 선택 시 해당 솔루션을 구매한 고객사의 등록 이메일 전체로 자동 발송

**직접 입력 방식**
- 수신자 이메일을 직접 입력 (여러 명 추가 가능)

### 발송 절차

1. 발송 방법 선택 (솔루션 / 직접 입력)
2. 제목 입력
3. 본문 작성
4. **미리보기** 버튼으로 실제 이메일 렌더링 확인
5. **수신자 확인** 버튼으로 수신 대상 목록 확인
6. **발송** 클릭

발송 결과는 **발송 로그** 메뉴에서 확인합니다.

## 10. 공문 템플릿 설정 (Notice Settings)

**권한:** Admin, Manager

**경로:** 사이드바 > Management > Notice Settings

공문 이메일의 로고, 헤더 색상, 하단 문구를 커스터마이징합니다.

| 항목 | 설명 |
|------|------|
| 상단 로고 | 이메일 상단에 표시할 이미지 파일 (PNG/JPG) 및 너비(px) |
| 하단 로고 | 이메일 하단 푸터에 표시할 이미지 파일 및 너비(px) |
| 헤더 색상 | 카드 상단 강조 라인 색상 (컬러 피커 또는 hex 직접 입력) |
| 하단 문구 | 이메일 하단에 표시되는 안내 문구 (줄바꿈 적용) |

1. 각 항목 설정 후 **저장** 클릭
2. 우측 미리보기에서 실제 이메일 레이아웃 확인
3. **크게 보기** 버튼으로 모달에서 상세 확인 가능

> 설정값은 DB에 저장되어 서버 재시작 없이 즉시 반영됩니다.

## 11. Slack 워크스페이스 관리

**권한:** Admin, Manager

**경로:** 사이드바 > Slack 관리

고객사가 Slack 봇을 설치하면 이 화면에 워크스페이스가 나타납니다.

### 승인 절차

1. 새 워크스페이스가 **대기 중** 상태로 표시되면 확인
2. 해당 워크스페이스의 **승인** 버튼 클릭
   - 승인 후 고객사 담당자가 Slack Home Tab을 사용할 수 있음
   - 필요 시 **거부** 처리 가능

### 고객사 연결

승인 후 워크스페이스와 고객사를 연결합니다.

1. 워크스페이스 우측 **고객사 연결** 클릭
2. 드롭다운에서 고객사 선택
3. **저장**

연결이 완료되면 고객사 담당자가 Slack Home Tab에서 구독 설정을 직접 관리할 수 있습니다.

### 고객사 담당자에게 설치 링크 안내

고객사 담당자에게 아래 링크를 전달합니다:

```
https://slack-api.yourdomain.com/slack/install/
```

## 12. Notion 동기화

**권한:** Admin, Manager

**경로:** 사이드바 > Notion

Notion 페이지의 패치노트 내용을 DB와 동기화합니다.

Notion Integration 페이지는 **Product**와 **Utility** 탭으로 분리되어 있습니다.

### 페이지 매핑 등록

**Product 탭**
1. **매핑 추가** 클릭
2. 제품 선택
3. Notion 페이지 ID 입력
   - Notion 페이지 URL의 마지막 32자리 문자열
   - 예: `https://notion.so/팀명/페이지명-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`
4. 한국어 / 영문 페이지 각각 입력 (영문은 선택사항)
5. **저장**

**Utility 탭**
1. **매핑 추가** 클릭
2. 유틸리티 선택
3. Notion 페이지 ID 입력
4. **저장**

> Utility Notion 페이지에는 `# 지원중인 버전` 섹션이 있어야 새 버전 Push가 가능합니다.

### 동기화

| 버튼 | 방향 | 설명 |
|------|------|------|
| **동기화** | Notion → DB | Notion에서 수정한 내용을 DB에 반영 |
| **푸시** | DB → Notion | DB 패치노트 내용을 Notion 페이지에 업데이트 |

- 동기화는 Notion 페이지의 마지막 수정 시각을 비교하여 변경이 있을 때만 반영됩니다.

## 13. 발송 로그 확인하기

**권한:** Admin, Manager

**경로:** 사이드바 > 발송 로그

### 필터 옵션

## 14. Action Log

**권한:** Admin, Manager

**경로:** 사이드바 > Management > Action Log

웹 UI에서 수행된 주요 액션 이력을 조회합니다. 발송 로그와 달리 **누가 어떤 작업을 했는지** 추적하는 감사 로그입니다.

### 기록되는 액션

| 분류 | 액션 |
|------|------|
| 로그인 | 로그인 성공 / 로그인 실패 |
| 패치노트 | 등록 / 수정 / 삭제 / 발행 |
| Notion | 동기화 / Push |
| 고객사 | 등록 / 수정 / 삭제 |
| 구독 | 구독 설정 변경 |
| Slack | 워크스페이스 승인 / 거부 |
| 솔루션 / 제품 | 등록 / 수정 / 삭제 |
| 유틸리티 | 등록 / 수정 / 삭제 |
| 공문 | 발송 |
| 공문 템플릿 | 설정 변경 |

### 로그 항목 확인 정보

- 수행자 (계정명)
- 액션 유형
- 대상 (패치노트 버전, 고객사명 등)
- 상세 정보 (등록 내용, 발송 결과 등)
- IP 주소
- 일시
