## 1. 로그인 및 화면 구성

로그인 후 대시보드가 표시됩니다. 왼쪽 사이드바에서 원하는 메뉴로 이동합니다.

**사이드바 메뉴 구성**

- **솔루션 > 제품** — 클릭하면 해당 제품의 패치노트 목록으로 이동
- **고객사 관리** — 고객사 등록 및 이메일 관리
- **구독 설정** — 고객사별 Gmail/Slack 구독 설정
- **공문 발송** — 공지사항 이메일 발송
- **Slack 관리** — Slack 워크스페이스 승인 및 고객사 연결
- **Notion** — Notion 페이지 동기화
- **발송 로그** — 이메일/Slack 발송 이력 조회

> 역할에 따라 보이는 메뉴가 다를 수 있습니다.

---

## 2. 패치노트 작성하기

**권한:** Admin, Manager, Dev

1. 왼쪽 사이드바에서 **솔루션 > 제품명** 클릭
2. 우측 상단 **패치노트 추가** 버튼 클릭
3. 버전과 배포일 입력
4. 아래 항목에 내용 입력:

| 항목 | 설명 |
|------|------|
| 기능 추가 | 이번 버전에서 새로 추가된 기능 |
| 기능 개선 | 기존 기능이 변경되거나 향상된 내용 |
| 버그 수정 | 수정된 버그 목록 |
| 특이사항 | 주의사항, 마이그레이션 안내 등 |
| 내부 사항 | 외부 발송에 포함되지 않는 내부 메모 |

5. 각 항목에서 **+ 추가** 버튼으로 항목 추가, 하위 항목 버튼으로 들여쓰기 항목 추가 가능
6. **저장** 클릭

> Ollama 번역 서버가 연결되어 있으면 저장 후 영문 번역이 자동으로 시작됩니다.
> 기능 추가, 기능 개선, 버그 수정의 경우 기존과 동일하게 CodeBlock안에 작성되기 때문에 평문으로 작성 부탁드립니다.

---

## 3. 패치노트 수정 및 삭제

- **수정:** 패치노트 항목 클릭 → 내용 수정 → **저장**
- **삭제:** 패치노트 우측 삭제 버튼 클릭 → 확인

> 이미 발행된 패치노트는 수정해도 기존에 발송된 알림이 다시 전송되지 않습니다.

---

## 4. 파일 첨부하기

**권한:** Admin, Manager, Dev

패치노트에 배포 파일(APK, AAR, Framework 등)을 첨부할 수 있습니다.

1. 패치노트 상세 화면 하단 **파일 첨부** 영역으로 이동
2. 파일 유형 선택:
   - **Release** — 정식 배포 파일
   - **Debug** — 디버그 정보 포함 파일
3. 파일 선택 후 **업로드**
4. Nextcloud(NAS)가 연결된 경우 자동으로 이중 저장되며 공유 링크가 생성됩니다.

---

## 5. 패치노트 발행하기

**권한:** Admin, Manager, Dev

> ⚠️ 발행하면 즉시 알림이 발송됩니다. 내용을 꼭 확인하고 발행하세요.

1. 발행할 패치노트 우측 **발행** 버튼 클릭
2. 확인 다이얼로그에서 **발행** 클릭
3. 발행이 완료되면 구독 중인 고객사에 알림이 자동 발송됩니다:
   - **Gmail** 구독자 → 이메일로 전송
   - **Slack** 구독자 → 지정된 채널로 메시지 전송

**발송되는 Slack 메시지 예시**

```
[AppSuit Android Library Release 안내]

Version: 1.2.3  ·  2025-01-15
──────────────────────────────
기능 추가
  - 신규 API 지원
버그 수정
  - 크래시 수정
```

발송 결과는 **발송 로그** 메뉴에서 확인할 수 있습니다.

## 6. 고객사 관리하기

**권한:** Admin, Manager, SE

**경로:** 사이드바 > 고객사 관리

### 고객사 등록

1. **고객사 추가** 버튼 클릭
2. 고객사명 입력
3. On-Premise 여부 체크 (해당되는 경우)
4. 구매 솔루션 선택 (복수 선택 가능)
5. **저장**

### 수신 이메일 추가

1. 고객사 카드 내 **이메일 추가** 클릭
2. 이메일 주소와 담당자명(선택) 입력
3. **추가**

여러 담당자 이메일을 등록하면 패치노트 발행 시 전원에게 발송됩니다.

### Google 연락처 CSV로 일괄 추가

1. Google 연락처에서 CSV 내보내기
2. **CSV 가져오기** 버튼으로 파일 업로드
3. 이메일 자동 등록

## 7. 구독 설정하기

**권한:** Admin, Manager, SE

**경로:** 사이드바 > 구독 설정

고객사별로 어떤 제품의 패치노트를 어떤 방식으로 받을지 설정합니다.

1. 고객사 선택
2. 제품별로 **Gmail** / **Slack** 활성화 여부 설정
3. **최대 수신 건수** 설정 (1~10건)
   - 발행 시 최근 N건의 패치노트를 묶어서 전송
4. Slack 알림의 경우 **채널 ID** 입력

   > **채널 ID 확인 방법:** Slack 채널명 우클릭 → 채널 정보 보기 → 하단에 채널 ID 표시

5. **저장**

## 8. 공문 발송하기

**권한:** Admin, Manager

**경로:** 사이드바 > 공문 발송

패치노트 알림과 별개로 공지사항이나 안내문을 이메일로 발송합니다.

### 발송 방법 선택

**솔루션 선택 방식**
- 솔루션 선택 시 해당 솔루션을 구매한 고객사의 등록 이메일 전체로 자동 발송

**직접 입력 방식**
- 수신자 이메일을 직접 입력 (여러 명 추가 가능)

### 발송 절차

1. 발송 방법 선택 (솔루션 / 직접 입력)
2. 제목 입력
3. 본문 작성
4. **미리보기** 버튼으로 실제 이메일 렌더링 확인
5. **수신자 확인** 버튼으로 수신 대상 목록 확인
6. **발송** 클릭

발송 결과는 **발송 로그** 메뉴에서 확인합니다.

## 9. Slack 워크스페이스 관리

**권한:** Admin, Manager

**경로:** 사이드바 > Slack 관리

고객사가 Slack 봇을 설치하면 이 화면에 워크스페이스가 나타납니다.

### 승인 절차

1. 새 워크스페이스가 **대기 중** 상태로 표시되면 확인
2. 해당 워크스페이스의 **승인** 버튼 클릭
   - 승인 후 고객사 담당자가 Slack Home Tab을 사용할 수 있음
   - 필요 시 **거부** 처리 가능

### 고객사 연결

승인 후 워크스페이스와 고객사를 연결합니다.

1. 워크스페이스 우측 **고객사 연결** 클릭
2. 드롭다운에서 고객사 선택
3. **저장**

연결이 완료되면 고객사 담당자가 Slack Home Tab에서 구독 설정을 직접 관리할 수 있습니다.

### 고객사 담당자에게 설치 링크 안내

고객사 담당자에게 아래 링크를 전달합니다:

```
https://slack-api.yourdomain.com/slack/install/
```

## 10. Notion 동기화

**권한:** Admin, Manager

**경로:** 사이드바 > Notion

Notion 페이지의 패치노트 내용을 DB와 동기화합니다.

### 페이지 매핑 등록

1. **매핑 추가** 클릭
2. 제품 선택
3. Notion 페이지 ID 입력
   - Notion 페이지 URL의 마지막 32자리 문자열
   - 예: `https://notion.so/팀명/페이지명-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`
4. 한국어 / 영문 페이지 각각 입력 (영문은 선택사항)
5. **저장**

### 동기화

| 버튼 | 방향 | 설명 |
|------|------|------|
| **동기화** | Notion → DB | Notion에서 수정한 내용을 DB에 반영 |
| **푸시** | DB → Notion | DB 패치노트 내용을 Notion 페이지에 업데이트 |

- 동기화는 Notion 페이지의 마지막 수정 시각을 비교하여 변경이 있을 때만 반영됩니다.

## 11. 발송 로그 확인하기

**권한:** Admin, Manager

**경로:** 사이드바 > 발송 로그

이메일과 Slack 발송 이력 전체를 조회합니다.

### 필터 옵션

| 필터 | 선택지 |
|------|--------|
| 발송 유형 | 공문 / 구독 자동 발송 |
| 채널 | Gmail / Slack |
| 상태 | 성공 / 실패 / 대기 |
| 고객사 | 전체 또는 특정 고객사 |
| 날짜 | 기간 지정 |

### 로그 항목 확인 정보

- 고객사명
- 수신자 (이메일 주소 또는 Slack 채널 ID)
- 발송 제목
- 발송 상태
- 실패 시 오류 메시지
- 발송 일시
